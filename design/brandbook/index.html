02 / Principles

Quiet structure.
Visible agency.

Trace feels dependable because the interface tells the truth: who owns control, what will happen, where files live, and how to recover.

01

Make ownership visible

Control, presence, and editability are persistent state—not surprise messages after an action.

02

Protect the work plane

Chrome is compact and quiet. Color carries state and focus; content keeps the visual priority.

03

Structure before decoration

Rails, frames, alignment, and spacing establish rhythm. Effects are restrained and purposeful.

04

Motion explains change

Transitions connect cause and effect. Reduced-motion behavior preserves clarity without movement.

10 / Voice

Calm precision.

Trace speaks like a capable collaborator: direct, technically honest, privacy-aware, and explicit about ownership, control, files, failure, and recovery.

Direct

Lead with the state or action. Remove ceremony and filler.

Honest

Name what happened, what is known, and what remains uncertain.

Protective

State where files live and when data leaves the device.

Recoverable

Offer the safest useful next step without blaming the person.

Use

Aria has control. You can keep viewing the workspace.

We couldn’t push. Your commit is still saved locally.

Discard 3 unstaged changes in App.tsx?

Avoid

You do not have permission to edit.

Oops! Something went wrong.

Are you sure you want to proceed?

Actions

Use verb + object: “Request control”, “Open local folder”, “Retry push”.

Privacy

Name the boundary: “Stays on this device” or “Shared with workspace members”.

Errors

State effect before cause: “Push failed. Your commit remains local.”

Destructive

Name count and target. Explain recovery before confirmation.

11 / Accessibility & adoption

Clarity is part
of the system.

Production text targets 4.5:1; graphical controls and large text target 3:1. Keyboard, screen-reader, zoom, and reduced-motion behavior are requirements, not polish.

Implemented

Documented reality

Current Trace surfaces, neutral modes, compact dimensions, syntax roles, accents, and motion values are transcribed from the application.

Extension

Approved guidance

Canonical Frame usage, fixed brand palette, graphic language, darker solid-action colors, display type, and voice rules extend the system.

Migration target

Known contrast gaps

Muted and faint light-mode text, plus some base-accent solid actions, need accessible token adoption. Do not treat documentation as a silent runtime fix.

Scope boundary

This book documents; it does not migrate.

No application components, runtime styles, onboarding flow, packaging, or dependencies are changed here. App icons, social templates, legacy in-app mark replacement, shared token extraction, and component migration are future adoption work.
